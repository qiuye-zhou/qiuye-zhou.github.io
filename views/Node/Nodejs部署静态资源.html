如何搭建服务器部署静态网站

秋叶 5/31/2022 VueNode项目

本地项目搭建完成后，怎么将项目发布到服务器？

# 通过Node创建服务器

关键

创建Node项目，通过express快速创建web服务器，将vue打包生成的dist文件（或者其他打包后生成的文件）托管为静态资源就成功部署了

首页要安装Node环境 可以去Nodejs中文网 (opens new window)下载安装

然后安装express并将其保存到依赖中

$ cnpm install express --save
1

同时安装好下面几个重要的模块

$ cnpm install body-parser --save
$ cnpm install cookie-parser --save
$ cnpm install multer --save
1
2
3

新建一个文件夹，这里我命名为:node_server

进入该目录，npm init -y 初始化项目

创建server.js文件

//安装express后引入express模块
var express = require('express');

//创建web服务器
const serve = express();

//托管静态资源
serve.use(express.static('./dist'))

//启动web服务器
serve.listen(8086, () => {
    console.log('server up : http://127.0.0.1:8086');
    console.log('server up : http://localhost:8086');
})

# 打包项目部署

将需要部署到服务器的项目进行打包，输入 npm run build 或者 yarn run build，打包项目

将打包后生成的dist文件夹，复制到node_server目录下

完成后的文件结构应该是这样的

  • node_server

    • dist

    • node_modules

    • package.json

    • server.js

最后使用node server.js运行服务器，终端打印出

server up : http://127.0.0.1:8086

server up : http://localhost:8086

则说明服务成功启动，前往浏览器访问就可以看到部署的项目了。

# 使用gzip文件压缩，提升访问速度

gzip概述

HTTP协议上的gzip编码是一种用来改进web应用程序性能的技术。 即指在www服务器上安装一个功能，实现当有人访问服务器时，这个功能将网页内容压缩后传输到访客浏览器中显示(web服务器和客户端（浏览器）必须共同支持gzip。目前主流的浏览器，都支持该协议)，对纯文本内容能压缩到原来的40%左右，这样传输包的大小变小了，使得传输时间变短了，提升了访问速度。

安装：

npm i compression -S

安装完后修改server.js的内容

//安装express后引入express模块
var express = require('express');

//导入
const compression  = require("compression")

//创建web服务器
const serve = express();

//在托管静态资源之前，注册中间件
app.use(compression())

//托管静态资源
serve.use(express.static('./dist'))

//启动web服务器
serve.listen(8086, () => {
    console.log('server up : http://127.0.0.1:8086');
    console.log('server up : http://localhost:8086');
})
